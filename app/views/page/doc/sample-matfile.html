matfile例子

在快速开始章节已经给出过一个matfile例子，我们就拿那个例子做解释： 

var mat  = require('mat');
var rap  = require('mat-rap');
var opoa = require('mat-opoa');

mat.task('default', function () {
  mat.use(rap({
    projectId: "123"
  }))

  mat.use(opoa({
    root: './'
  }))
})

mat.task('daily', function () {
  mat.use(opoa({
    root: './',
    proxy: 'www.abc.net'
  }))
})

task

如果你用过glupjs，你一定对这种写法不陌生。mat的task和gulp的task是同一个概念，task用来定义不同的任务，mat具有执行任务的能力，默认情况下mat会使用default这个任务。

上面的例子中定义了两个不同的任务，代表了两种不同的应用场景：

  • default任务会使用mock数据
  • daily任务会使用反向代理取测试服务器数据

在项目的不同阶段跑不同的任务，快速切换。

use

加载mat插件

mat插件会按照加载的顺序执行，不同任务间的插件相互独立。
